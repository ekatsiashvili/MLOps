# Звіт порівняння Docker образів

## 1. Порівняння розмірів
| Тип образу | Тег | Розмір (фактичний) |
|------------|-----|--------------------|
| Fat        | fat | 3.22 GB            |
| Slim       | slim| 1.38 GB            |

## 2. Аналіз

### Fat Image
- **Розмір:** 3.22 GB.
- **Причина:** Встановлено повний `python:3.9` та PyTorch із підтримкою CUDA (для GPU), що займає найбільше місця. Також присутні системні інструменти (`git`, `curl`), які не потрібні для запуску.

### Slim Image
- **Розмір:** 1.38 GB.
- **Оптимізація:**
  1. Використано **Multi-stage build**: віртуальне середовище створено окремо.
  2. Використано **PyTorch CPU version**: це головний фактор зменшення розміру (~2 ГБ економії).
  3. Використано `python:3.9-slim`: менший базовий образ без зайвих системних бібліотек.

## 3. Висновки
Оптимізація дозволила зменшити розмір образу **в 2.3 рази**. Для сервісів, що працюють на CPU, використання спеціалізованих версій бібліотек (torch+cpu) та multi-stage збірки критично важливе.
